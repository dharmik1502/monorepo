Social Downloader — API Reference
For frontend developers · Version 1.0
POST /api/v1/download
01 Overview

Single endpoint. Send the social media post URL, receive direct download links — video, image, or audio. Authentication is optional; unauthenticated requests work but share a rate limit.

All requests and responses use JSON. Base URL in production is your Vercel domain. In local dev: http://localhost:3000.
02 Request Headers

Two headers — one always required, one only when the user is logged in.

Header Value When Description
Content-Type application/json Required Must be set on every request. Without it the server cannot parse the body.
Headers
Content-Type: application/json
03 Request Body

Send a JSON object. Only url is required.

Field Type Description
url string Required Full URL of the social media post including https://.
Supported: Instagram, Facebook, Twitter/X, TikTok, Pinterest
quality "hd" | "sd" Optional Preferred quality. If unavailable, server returns the best it has.
Default: best available
Request body — JSON
{
  "url":     "https://www.instagram.com/p/ABC123/",
  "quality": "hd"   // optional
}
04 Response Shape

HTTP 200 always. Check success first — even when media extraction fails the status is 200, so a false here means no download links were found.

Response — success: true
{
  "success":  true,

  "metadata": {
    "platform":    "instagram",
    "title":       "Sunset at the beach",
    "description": "Golden hour vibes...",
    "thumbnail":   "https://cdn.instagram.com/thumb.jpg",
    "author":      "@username",
    "duration":    42   // seconds; null for images
  },

  "urls": [
    {
      "url":        "https://cdn.instagram.com/video.mp4",
      "quality":   "hd",
      "type":      "video",   // "video" | "image" | "audio"
      "extension": "mp4",
      "size":      4194304   // bytes; may be null
    }
  ]
}
Field in urls[]TypeAlways presentDescription
url string Yes Direct download link. These expire — download or proxy promptly.
type "video" | "image" | "audio" Yes Use this to decide whether to render a video player or an image tag.
quality string No e.g. "hd", "sd", "1080p". Null for images.
extension string No File extension without dot — mp4, jpg, mp3.
size number No File size in bytes. Check for null before displaying.
05 Code Examples
JavaScript — fetch()
const download = async (url) => {
  const headers = { 'Content-Type': 'application/json' };

  const res = await fetch('/api/v1/download', {
    method: 'POST',
    headers,
    body: JSON.stringify({ url }),
  });

  if (!res.ok) throw new Error(`HTTP ${res.status}`);

  const data = await res.json();
  if (!data.success) throw new Error('No media found');

  return data.urls;  // array of download items
};
JavaScript — axios
import axios from 'axios';

const download = async (url) => {
  const { data } = await axios.post('/api/v1/download', { url }, {
    headers: { 'Content-Type': 'application/json' },
  });

  if (!data.success) throw new Error('No media found');

  return data.urls;
};
cURL
curl -X POST https://your-domain.vercel.app/api/v1/download \
  -H "Content-Type: application/json" \
  -d '{"url": "https://www.tiktok.com/@user/video/123"}'
06 Error Responses

Non-2xx means the server rejected the request. Different from success: false (which is a 200 with no media found).

400
Bad Request — URL from an unsupported platform, or missing/invalid url field.
{"statusCode": 400, "message": "Unsupported platform. Supported: Instagram, Facebook, Twitter/X, TikTok, Pinterest."}
429
Too Many Requests — 10 requests/minute limit exceeded. Wait 60 seconds and retry.
{"statusCode": 429, "message": "ThrottlerException: Too Many Requests"}
401
Unauthorized — Authorization header present but token is invalid or expired.
{"statusCode": 401, "message": "Unauthorized"}
Note: A 200 with "success": false is NOT an error — the server worked but could not extract media from that URL. Show the user a friendly "No media found" message and let them try a different link.
07 Supported Platforms

Any URL not matching these returns 400. Validation happens before any network call.

Instagram Facebook Twitter / X TikTok Pinterest
08 Rate Limiting
10
requests per minute
per IP address

Hitting the limit returns 429. The window resets after 60 seconds. If your UI lets users submit multiple links at once, queue them client-side to avoid hitting the limit.
